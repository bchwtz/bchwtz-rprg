Aufgabe 4

Eine ist eine endliche Folge \(s = \left(s_1, s_2, ..., s_n\right)\) mit \(s_j \in \{-1, +1\}\) für alle \(1 \leq j \leq n\). Für \(0 < k < n\) ist die (der Ordnung k) definiert durch:

\begin*{equation} C_k(s) = {j=1}^{n-k} s_j s{j+k} \end{equation}

In R kann \(s\) leicht durch einen Integer-Vektor dargestellt und die aperiodische Autokorrelation bei-spielsweise wie folgt implementiert werden:

autokorrelation1 <- function(s) {
  n <- length(s)
  res <- vector(mode = "integer", length = n - 1)
  for (k in 1:(n-1)) {
    res[k] <- as.integer(t(s[1:(n - k)]) %*% s[(k + 1):n])
  }
  return(res)
}

  1. Modifizieren Sie die Funktion autokorrelation1(), indem Sie die Schleife durch map_int() (aus purrr) ersetzen. Nennen Sie die modifizierte Version autokorrelation2().

  2. Vergleichen Sie die Laufzeiten der beiden Funktionen für große n mithilfe von system.time(), beispielsweise durch:

randBinSeq <- function(n) {
  return(sample(c(-1L, 1L), n, replace = TRUE))
}
s <- randBinSeq(2E4) 
autokorrelation1(s)
autokorrelation2(s)
  1. Erstellen Sie einen weiteren Laufzeittest; erklären und interpretieren Sie die Ergebnisse:
require(compiler)
f <- cmpfun(autokorrelation1)
system.time(f(s))

Aufgabe 6

Erklären Sie den folgenden Quelltext:

foo1 <- function(x,...) {
  print(x)
  args <- list(...)
  for(a in args){
    print(a)
  }
}

foo1(1)
foo1(1,2,3,4, mean=6)
foo2 <- function(n, ...) {
  return(rnorm(n, ...))
}

foo2(4)
foo2(4,100)
foo2(4, mean=100)
foo2(4, mean=100, sd=500)
foo2(4, meen=100, sd=500)
foo3 <- function(x, ...) {
  return(c(mean(x,...), median(x,...)))
}

x <- c(1,2,100,4)
foo3(x)
x <- c(1,2,NA, 100,4)
foo3(x)
foo3(x, na.rm=TRUE)
foo3(x, rm.na=TRUE)
