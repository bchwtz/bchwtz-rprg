7.1.1 Boxplots

Inhaltliche Voraussetzung: Boxplot, unteres Quartil, Median, oberes Quartil

Führen Sie folgenden Quelltext aus:

anzahl <- 8
op <- par(mfrow=c(2,anzahl/2), col="blue") # Grafikeinstellung ändern
n <- 100L
x <- 0:n 
for (i in 0:(anzahl-1)){
  x[n] <- x[n]+10*i
  x[n-1] <- x[n-1]+5*i
  print(paste("i = ",i, ":")) 
  print(summary(x)) 
  boxplot(x, main = paste("i = ",i),   
        xlab = paste("x[n-1] =", x[n-1], ", x[n] =", x[n]))
}
par(op)  # stellt die ursprüngliche Grafikeinstellung wieder her

Erklärungen zum Quelltext:

  • Durch par() können Grafikoptionen gesetzt werden. In unserem Fall werden die nächsten 8 Bilder in einem Plot bestehend aus 2×4 Einzelbildern dargestellt.

  • paste() konvertiert die übergebenen Argumente zu Zeichenketten (character-Vektoren) und fügt sie zusammen.

plotZiffer <- function(ziffern, zeile){
  m <- matrix(ziffern[zeile,], nrow = 28, ncol = 28, byrow = T)
  plot(as.raster(m, max = 255))
}
par(mfrow = c(3, 3))
for(z in 1:9){ # Plotten der ersten neun Ziffern
  plotZiffer(ziffern, z)
}

Figure 7.2: Plots der ersten neun Ziffern aus dem Ziffern-Trainingsdatensatz (Irizarry and Gill 2021).

Vor die aufgelistete R-Bezeichnung (wie norm) kommt noch ein entsprechender Buchstabe (Die folgenden Beipiele beziehen sich auf die Normalverteilung.):

  • d für die Dichte. Beispiel: dnorm(x)

  • p für die Verteilungsfunktion. Beispiel: pnorm(x)

  • q für die Quantilfunktion. Beispiel: qnorm(p)

7.3.2 Funktionsplot

In einer ggplot2-Grafik kann leicht mit stat_function() ein Funktionsplot in einer neuen Ebene hinzugefügt werden. Die zu zeichnende Funktion wird dabei durch das Argument fun bestimmt.

ggplot(data.frame(x = c(-4, 4)), aes(x)) +
  stat_function(fun = dnorm, colour = "red")

Figure 7.4: Dichte der Standardnormalverteilung N(0;1).

Statt dnorm hätten wir auch eine andere Funktion wie z.B. sin oder den Bezeichner einer selbst geschriebenen numerischen Funktion angeben können. Ähnlich leicht können mehrere Ebenen in einer Grafik gleichzeitig angezeigt werden. Als Beispiel verwenden wir die Dichten der \(\chi^2\)-Verteilung mit unterschiedlichen Freiheitsgraden:

?dchisq 
sizeline <- 1
ggplot(data.frame(x = c(0, 10)), aes(x)) + 
  stat_function(fun = dchisq, args = list(df=1), color = "cyan", size = sizeline) +
  stat_function(fun = dchisq, args = list(df=2), color = "red", size = sizeline) +
  stat_function(fun = dchisq, args = list(df=3), color = "blue", size = sizeline)+
  stat_function(fun = dchisq, args = list(df=4), color = "green3", size = sizeline)+
  labs(title = "Dichten der Chi-Quadrat-Verteilungen", subtitle ="Data Science, J. Willms")

Figure 7.5: \(\chi^2\)-Verteilung mit den Freiheitsgraden \(df = \{1, 2, 3, 4\}\).

Wie wir sehen, können in stat_function() zusätzliche Argumente für den Funktionsaufruf mithilfe des Arguments args in Form einer Liste übergeben werden. Das nächste Beispiel zeigt die Dichten der \(t\)-Verteilung :

sizeline <- 1
ggplot(data.frame(x = c(-4, 4)), aes(x)) + 
  stat_function(fun = dt, args = list(df=1), color = "cyan", size = sizeline) +
  stat_function(fun = dt, args = list(df=2), color = "red", size = sizeline) +
  stat_function(fun = dt, args = list(df=3), color = "blue", size = sizeline)+
  stat_function(fun = dt, args= list(df=4), color = "green3", size = sizeline)+
  labs(title = "Dichten der t-Verteilungen", subtitle ="Data Science, J. Willms") +
  theme_light()

Figure 7.6: Dichte der \(t\)-Verteilung mit den Freiheitsgraden \(df = \{1, 2, 3, 4\}\).

Im letzten Beispielplot haben wir ein anderes „Theme“ benutzt. ggplot2() unterstützt standardmäßig acht unterschiedliche „Themes“, mit denen die globalen Eigenschaften der Nicht-Daten-Elemente in einer Grafik gesetzt werden können (siehe ?theme und ?theme_bw).

7.3.3 Histogramme

Wir beginnen mit einem einfachen Datensatz x. Wir haben bereits gesehen, dass mithilfe der Funktion hist() (Bestandteil von Base-R) ein Histogramm erstellt werden kann:

set.seed(1)
x = rnorm(50)
h <- hist(x, breaks = 10)

Figure 7.7: Histogramm des Datensatzes x mittels hist() erzeugt.

Als Nächstes wollen wir mithilfe von ggplot() ein ähnliches Histogramm erstellen und dann nacheinander weitere Ebenen in die Grafik bringen:

library(ggplot2)
ggplot(data.frame(x = x), aes(x)) + 
  geom_histogram(bins=9)   # y: count

Figure 7.8: Histogramm des Datensatzes x mittels ggplot2 erzeugt.

Überraschenderweise hat das so erzeugte Histogramm eine andere Form. Der Grund hierfür ist, dass die beiden Histogramme auf unterschiedlichen Gruppenaufteilungen basieren. Die Gestalt eines Histogramms hängt sehr wesentlich von der Anzahl der Gruppen („bins“), der einzelnen Gruppenbreiten sowie von der unteren Grenze der ersten Gruppe (bzw. von der oberen Grenze der letzten Gruppe) ab. Im Folgenden wollen wir dies anhand von Beispielen illustrieren.

Mithilfe von str(h) sehen wir, wie die Gruppen im ersten Histogramm gebildet worden sind.

str(h)
## List of 6
##  $ breaks  : num [1:10] -2.5 -2 -1.5 -1 -0.5 0 0.5 1 1.5 2
##  $ counts  : int [1:9] 1 1 2 6 14 8 13 3 2
##  $ density : num [1:9] 0.04 0.04 0.08 0.24 0.56 0.32 0.52 0.12 0.08
##  $ mids    : num [1:9] -2.25 -1.75 -1.25 -0.75 -0.25 0.25 0.75 1.25 1.75
##  $ xname   : chr "x"
##  $ equidist: logi TRUE
##  - attr(*, "class")= chr "histogram"

Den gleichen Effekt erhalten wir, wenn wir in geom_histogram() als Grenze boundary = -2.5 und als Gruppenbreite binwidth = 0.5 angeben und zusätzlich theme_minimal() verwenden:

ggplot(data.frame(x = x), aes(x)) + 
  geom_histogram(boundary=-2.5, binwidth = 0.5, fill="white", color="black") + 
  theme_minimal()

Figure 7.9: Angepasstes Histogramm, mittels ggplot2 erstellt.

7.3.3.1 Unterschiedliche Anzahl der Gruppen

Es ist eigentlich nicht überraschend, dass die Gestalt eines Histogramms bei identischen Daten sehr wesentlich von der Anzahl der Gruppen („bins“) abhängt.

Figure 7.10: Identische Daten, aber eine unterschiedliche Anzahl von Gruppen (“bins”) (bins = \(\{7, 8, 9, 10, 11, 12\}\)).

7.3.3.2 Unterschiedliche untere Grenzen

Überraschend ist vielleicht Folgendes: Selbst wenn die Anzahl der Gruppen und die Gruppenbreite konstant bleibt, kann die Gestalt eines Histogramms bei identischen Daten wesentlich von der unteren Grenze der ersten Gruppe (bzw. von der oberen Grenze der letzten Gruppe) abhängen!

Als Beispiel wollen wir einen bekannten Datensatz verwenden, der die Schneehöhen in Inches aus 63 hintereinander folgenden Wintern auflistet, gemessen in Buffalo im Bundesstaat New York, USA (Eine detailliertere Behandlung dieses Beispiels finden Sie auch auf S. 11 im Abschnitt „Histogramme lassen sich weiter verfeinern“ (Kockelkorn 2012) oder (Arens et al. 2017, Kapitel 18, S. 206 ff.) (auch als EBook in der FH-SWF verfügbar, Stand Oktober 2019):

sHoehe <- c(25.0 , 39.8 , 39.9 , 40.1 , 46.7 , 49.1 , 49.6 , 51.1 , 51.6,
            53.5 , 54.7 , 55.5 , 55.9 , 58.0 , 60.3 , 63.6 , 65.4 , 66.1,

Die Gestalt der fünf Histogramme ist sehr unterschiedlich, obwohl alle auf denselben Daten basieren; auch die Anzahl der Gruppen und die Gruppenbreiten sind identisch! Dies wird auch deutlich, wenn die fünf Histogramme in einer Grafik als Überlagerung dargestellt werden:

transparenz <- 0.2
p <- ggplot(df, aes(x)) 
for(b in seq(16, 24, by=2)) {
  p <- p + geom_histogram(binwidth=10, boundary=b, alpha=transparenz) 
  transparenz <- transparenz+0.1
}
p + labs(title = "Überlagerung der fünf Histogramme")

7.3.3.3 Kerndichteschätzer geom_density()

In der Überlagerung der fünf Histogramme scheint nun eine Struktur erkennbar, die in den einzelnen Histogrammen nicht so ersichtlich ist. Hier helfen oft Kerndichteschätzer wie geom_density(), die das Histogramm glätten und ein stetiges Bild des Histogramms vermitteln.

Damit wir geom_density() anwenden können, müssen wir aes(x, stat(density)) statt aes(x) in ggplot() angeben. Statt der Anzahl (count) wird nun die Dichte (density) auf der y-Achse im Histogramm aufgetragen. Die Daten (Schneehöhen) übernehmen wir unverändert, ebenso die Gruppenbreite 10; die untere Grenze \(b\) der ersten Gruppe legen wir auf den Wert 16 fest:

ggplot(df, aes(x, stat(density))) + 
  geom_histogram(binwidth=10, boundary=16, alpha=0.2) +
  geom_density(color="blue", size=1)

Figure 7.13: (ref:geom_dens)

(ref:geom_dens) Histogramm mit geom_density().

Mit dem Argument adjust können wir die Dichteschätzung relativ zur Gruppenbreite anpassen; adjust = 0.5 nimmt die halbe Gruppenbreite als Grundlage der Schätzung:

ggplot(df, aes(x, stat(density))) + 
  geom_histogram(binwidth=10, boundary=16, alpha=0.2) +
  geom_density(color="blue", size=1)+
  geom_density(adjust=0.5, color="red", size =1)

Figure 7.14: (ref:geom_dens2)

(ref:geom_dens2) Histogramm mit geom_density() (rot: adjust = 0.5).

Aus meiner Sicht wird die Funktionsweise des Dichteschätzers geom_density() noch deutlicher, wenn man die Überlagerung der Histogramme zugrunde legt:

transparenz <- 0.2
p <- ggplot(df, aes(x, stat(density))) 
for(b in seq(16, 24, by=2)) {
  p <- p + geom_histogram(binwidth=10, boundary=b, alpha=transparenz) 
  transparenz <- transparenz+0.1
}
p <- p + geom_density(color="blue", size=1)+
    geom_density(adjust=0.5, color="red", size =1)
p + 
  theme_bw() +
  labs(title = "Überlagerung der Histogramme mit Dichteschätzer", 
         subtitle ="Data Science, J. Willms")

Figure 7.15: (ref:geom_dens3)

(ref:geom_dens3) Überlagerung und Kerndichteschätzer.

7.3.4 Zentraler Grenzwertsatz

Als nächstes Beispiel wollen wir die Konvergenzaussage des zentralen Grenzwertsatzes anhand eines elementaren Beispiels visualisieren. Abgesehen von dnorm(), der Dichtefunktion der Normalverteilung, wollen wir lediglich die Funktion sample() benutzen.

Als Erstes wollen wir einen zufälligen Wert x erzeugen, der mit Wahrscheinlichkeit 2/3 den Wert 0 und mit Wahrscheinlichkeit 1/3 den Wert 1 annimmt:

set.seed(1)
x1 <- sample(c(0,0,1),1)

Die (theoretische) Varianz der zugrundeliegenden zweiwertigen Zufallsvariablen \(X\) ist leicht berechnet: \(Var(X)= 2/3 \cdot 1/9 + 1/3 \cdot 4/9 = 2/9\).

zeichne <- function(df, nGruppen, n=300){
  supStr <- sprintf("anzahl=%d, n=300,  J. Willms", length(df$x)) 
  p <- ggplot(df, aes(x)) +
    geom_histogram(aes(x, stat(density)), bins=nGruppen, alpha=0.4)+
    geom_density(color="blue", size =1.2) +
    labs(title = "Konvergenz gegen die Normalverteilung?", subtitle = supStr) +
    stat_function(fun = dnorm, args= list(mean=1/3, sd=sqrt(2/(9*n))), 
                  colour = "red", size = 1.1) +
    theme_bw()
  return(p)
}

7.3.5 Quantilplot

Ein Quantilplot (siehe z.B. (Kockelkorn 2012, S. 173)) bietet die Möglichkeit, zwei Verteilungen auf Gleichheit visuell zu überprüfen. In einem Quantilplot werden die jeweiligen \(\alpha\)-Quantile zweier Verteilungen in einem Koordinatensystem gegeneinander aufgetragen. Liegen die gezeichneten Punkte annähernd auf einer Geraden, dann liegt die Vermutung nahe, dass es sich um ähnliche Verteilungen handelt. Wir wollen sogenannte \(NP-Plots\) benutzen, die eine Verteilung mit der Normalverteilung vergleichen.

Als einführendes Beispiel wollen wir einen Datensatz verwenden, von dem wir wissen, dass er N(4; 4)-verteilt ist.

Wir erhalten folgenden NP-Plot:

set.seed(0)  
s <- rnorm(400 ,mean=4, sd=2)  # N(4;4)-verteilt
ggplot(data.frame(s), aes(sample = s)) +
  stat_qq() + stat_qq_line()  +
  labs(title = "NP-PLot", subtitle = "N(4;4)-verteilt")

Figure 7.18: NP-Plot von N(4; 4) verteilten Daten.

Im Idealfall müssten alle Punkte auf einer Geraden liegen. Unser Beispiel ist „zufällig“ recht schlecht: Werden in unserem konkreten Beispiel andere „Seed“-Werte als 0 gewählt, ergeben sich sehr oft bessere Übereinstimmungen. Abweichungen finden sich im Allgemeinen häufig bei sehr kleinen oder sehr großen Quantil-Werten. Eine bessere Übereinstimmung erhält man in der Regel auch, wenn die Anzahl der Daten erhöht wird.

Aus dem y-Achsenabschnitt der eingezeichneten Geraden kann der Erwartungswert und aus der Steigung der Geraden kann die Streuung der theoretisch ermittelten Normalverteilung, die am „besten“ zu dem Datensatz passt, abgelesen werden. Unabhängig vom „Seed“ sollten in unserem Beispiel stets ungefähr 1/3 als Erwartungswert und 2 als Steigung abzulesen sein.
