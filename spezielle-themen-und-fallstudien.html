n <- 100L
x <- 0:n 
for (i in 0:(anzahl-1)){
  x[n] <- x[n]+10*i

Zur Erinnerung: Jedes Grauwertbild besteht dabei aus 28×28 Pixeln und jeder Pixelwert liegt zwischen 0 und 255.

ziffern <- mnist$train$images
dim(ziffern) # 60.000 Bilder (28*28 = 784 Pixel pro Bild)
## [1] 60000   784

Mithilfe folgender Funktion können die ersten neun Ziffern wie folgt geplottet werden:

# Funktion, um die einzelnen Ziffern zu plotten
plotZiffer <- function(ziffern, zeile){
  m <- matrix(ziffern[zeile,], nrow = 28, ncol = 28, byrow = T)
  plot(as.raster(m, max = 255))
}
par(mfrow = c(3, 3))
for(z in 1:9){ # Plotten der ersten neun Ziffern
  plotZiffer(ziffern, z)
}

Figure 9.2: Plots der ersten neun Ziffern aus dem Ziffern-Trainingsdatensatz (Irizarry and Gill 2021).

9.3.2 Funktionsplot

In einer ggplot2-Grafik kann leicht mit stat_function() ein Funktionsplot in einer neuen Ebene hinzugefügt werden. Die zu zeichnende Funktion wird dabei durch das Argument fun bestimmt.

ggplot(data.frame(x = c(-4, 4)), aes(x)) +
  stat_function(fun = dnorm, colour = "red")

Figure 9.4: Dichte der Standardnormalverteilung N(0;1).

Statt dnorm hätten wir auch eine andere Funktion wie z.B. sin oder den Bezeichner einer selbst geschriebenen numerischen Funktion angeben können. Ähnlich leicht können mehrere Ebenen in einer Grafik gleichzeitig angezeigt werden. Als Beispiel verwenden wir die Dichten der \(\chi^2\)-Verteilung mit unterschiedlichen Freiheitsgraden:

?dchisq 
sizeline <- 1
ggplot(data.frame(x = c(0, 10)), aes(x)) + 
  stat_function(fun = dchisq, args = list(df=1), color = "cyan", size = sizeline) +
  stat_function(fun = dchisq, args = list(df=2), color = "red", size = sizeline) +
  stat_function(fun = dchisq, args = list(df=3), color = "blue", size = sizeline)+
  stat_function(fun = dchisq, args = list(df=4), color = "green3", size = sizeline)+
  labs(title = "Dichten der Chi-Quadrat-Verteilungen", subtitle ="Data Science, J. Willms")

Figure 9.5: \(\chi^2\)-Verteilung mit den Freiheitsgraden \(df = \{1, 2, 3, 4\}\).

Wie wir sehen, können in stat_function() zusätzliche Argumente für den Funktionsaufruf mithilfe des Arguments args in Form einer Liste übergeben werden. Das nächste Beispiel zeigt die Dichten der \(t\)-Verteilung :

sizeline <- 1
ggplot(data.frame(x = c(-4, 4)), aes(x)) + 
  stat_function(fun = dt, args = list(df=1), color = "cyan", size = sizeline) +
  stat_function(fun = dt, args = list(df=2), color = "red", size = sizeline) +
  stat_function(fun = dt, args = list(df=3), color = "blue", size = sizeline)+
  stat_function(fun = dt, args= list(df=4), color = "green3", size = sizeline)+
  labs(title = "Dichten der t-Verteilungen", subtitle ="Data Science, J. Willms") +
  theme_light()

Figure 9.6: Dichte der \(t\)-Verteilung mit den Freiheitsgraden \(df = \{1, 2, 3, 4\}\).

Im letzten Beispielplot haben wir ein anderes „Theme“ benutzt. ggplot2() unterstützt standardmäßig acht unterschiedliche „Themes“, mit denen die globalen Eigenschaften der Nicht-Daten-Elemente in einer Grafik gesetzt werden können (siehe ?theme und ?theme_bw).

library(ggplot2)
ggplot(data.frame(x = x), aes(x)) +

Überraschenderweise hat das so erzeugte Histogramm eine andere Form. Der Grund hierfür ist, dass die beiden Histogramme auf unterschiedlichen Gruppenaufteilungen basieren. Die Gestalt eines Histogramms hängt sehr wesentlich von der Anzahl der Gruppen („bins“), der einzelnen Gruppenbreiten sowie von der unteren Grenze der ersten Gruppe (bzw. von der oberen Grenze der letzten Gruppe) ab. Im Folgenden wollen wir dies anhand von Beispielen illustrieren.

ggplot(data.frame(x = x), aes(x)) +

9.5 Fallstudie 5: Shapiro-Wilk-Test und Kolmogorov-Smirnov-Test

9.5.1 Shapiro-Wilk-Test

In dieser Fallstudie wollen wir zuerst exemplarisch auf den Shapiro-Wilk-Test eingehen. Der Shapiro-Wilk-Test ist ein statistischer Signifikanztest, der die Hypothese überprüft, ob eine Normalverteilung vorliegt.

Die Nullhypothese \(H_0\) lautet: Die Daten der Grundgesamtheit sind normalverteilt. Die Alternativhypothese \(H_1\) lautet somit: Die Daten der Grundgesamtheit sind nicht normalverteilt.

Als Signifikanzniveau wählen wir zunächst \(\alpha = 0.05\) (Dies ist ein üblicher, aber trotzdem sehr willkürlicher Wert.).

Es ist in R sehr leicht, den Shapiro-Wilk-Test durchzuführen. Viel schwieriger ist es, die richtigen Schlüsse aus den berechneten Testergebnissen zu ziehen.

Zuerst wollen wir den Pseudo-Zufallsgenerator initialisieren:

set.seed(17122019)

Als Nächstes wollen wir eine Stichprobe bestehend aus 40 Elementen aus einer normalverteilen Grundgesamtheit testen:

stichprobe <- rnorm(40, mean = 5, sd = 3)
shapiro.test(stichprobe)
## 
##  Shapiro-Wilk normality test
## 
## data:  stichprobe
## W = 0.97987, p-value = 0.6846

Wie wir sehen, ist der p-Wert recht groß; wir haben also keinen Grund, die Nullhypothese \(H_0\) abzulehnen. Selbst ein großer p-Wert belegt nicht, dass \(H_0\) wahr ist. Dies Aussagekraft des Ergebnisses ist also in diesem Fall recht schwach.

Bei einer Stichprobe, die aus einer auf [0, 1] gleichverteilten Grundgesamtheit stammt, sollten die Testergebnisse anders aussehen:

stichprobe <- runif(40)
shapiro.test(stichprobe)
## 
##  Shapiro-Wilk normality test
## 
## data:  stichprobe
## W = 0.88576, p-value = 0.0007558

Da der p-Wert unter dem gewählten Signifikanzniveau \(\alpha\) liegt, ist die Nullhypothese \(H_0\) in diesem Fall abzulehnen. Wir können aber nicht ausschließen, dass \(H_0\) trotzdem richtig ist! Wird \(H_0\) aufgrund des Tests (fälschlicherweise) abgelehnt, obwohl \(H_0\) wahr ist (Fehler 1. Art), dann repräsentiert die Stichprobe ein sehr seltenes Ereignis, das höchstens mit Wahrscheinlichkeit \(\alpha\) eintritt. Daraus lässt sich aber nicht ableiten, dass die Wahrscheinlichkeit, dass \(H_0\) wahr ist, kleiner als das gewählte Signifikanzniveau \(\alpha\) ist (Leider wird dieser Schluss fälschlicherweise sehr oft gezogen.).

Im Folgenden sehen wir, dass die Funktion shapiro.test() eine Liste mit vier Elementen zurückgibt und dass p.value dem p-Wert entspricht:

sTest <- shapiro.test(stichprobe)

9.5.2 Funktionen zur mehrfachen Wiederholung des Shapiro-Wilk-Tests

Wir wollen als Nächstes für unterschiedliche Stichprobengrößen ähnliche Tests mehrfach wiederholen und jeweils untersuchen, wie oft die entsprechende Nullhypothese verworfen wird. Dafür verwenden wir vier Funktionen. Die erste Funktion werden wir benutzen, um unsere erhaltenen Ergebnisdaten zu visualisieren:

myPlot <- function(titel, nWerte, Anteil){
  g <- ggplot(data.frame(nWerte, Anteil), aes(x = nWerte, y = Anteil)) +
    ggtitle(titel, subtitle = 
              expression('H'[0]*': Normalverteilung liegt vor')) +
    ylab(expression('H'[0]*' verworfen')) + 
    xlab("Stichproben-Größe: n") + 
    scale_y_continuous(labels = scales::percent)  
  return(g)
}

Wir haben hier den Funktionsaufruf scale_y_continuous(labels = scales::percent) benutzt, um die y-Achse mit den entsprechenden Prozentzahlen zu skalieren. Neu ist auch expression('H'[0]*': Normalverteilung liegt vor')). Damit wird der folgende Text im Plot erzeugt: Der mit [] umschlossene Buchstabe wird also tiefergestellt (Um einen Buchstaben hoch zu stellen, wird ^ statt [] benutzt.). Auf die genaue Syntax wollen wir an dieser Stelle nicht eingehen, sondern lediglich ein weiteres Beispiel anführen, das in Abbildung 9.21 resultiert:

                              ' und x' ^ 'n'))

Figure 9.21: Leerer Plot mit tiefer und höher gestellten Zahlen und Buchstaben im Titel.

Die zweite FunktionpShapiro(n) erzeugt in [-1,1] gleichverteilte Stichproben der Größe n, führt den Shapiro-Wilk-Test durch und gibt den entsprechenden p-Wert zurück. Wollen wir eine andere Wahrscheinlichkeitsverteilung zugrunde legen, dann muss diese Funktion angepasst werden.

# Shapiro-Wilk-Test
# Stichproben gleichverteilt
# n: Stichprobengröße
pShapiroTest <- function(n){
  stichprobe <- runif(n, min=-1, max=1)
  h <- shapiro.test(stichprobe) 
  return(h$p.value)
}

Die dritte Funktion replicateTest() ruft nTest-mal die übergebene Funktion FUN(n) auf und gibt den Anteil zurück, wie oft der entsprechende p-Wert unter dem Signifikanzniveau \(\alpha\) gelegen hat.

# n: Stichprobengröße
# FUN: Testfunktion (Default: pShapiroTest)
# nTest: Anzahl der Tests (Wiederholungen) 
# alpha: entspricht dem Signifikanzniveau
replicateTest <- function(n, FUN = pShapiroTest, nTest, alpha){
  res <- replicate(nTest, FUN(n))
  return(sum(res <= alpha)/nTest)
}

In der vierten Funktion testShapiro() wird schließlich

  • die Stichprobengröße n,

  • welcher Test benutzt werden soll (der Defaultwert ist die Funktion pShapiroTest()),

  • wie oft (bei gegebener Stichprobengröße) ein Test ausgeführt werden soll (nTest) und

  • das Signifikanzniveau alpha

festgelegt. Außerdem wird die Funktion replicateTest() aufgerufen:

testShapiro <- function(n){
  replicateTest(n, FUN=pShapiroTest, nTest=1000, alpha=0.05)
}

9.5.3 Testgüte des Shapiro-Wilk-Tests bei Gleichverteilung

Nun können wir sehr leicht die Testgüte des Shapiro-Wilk-Tests untersuchen und die Ergebnisse visuell darstellen. Nach wie vor lautet die Nullhypothese \(H_0\): Die Daten der Grundgesamtheit sind normalverteilt. Die Alternativhypothese \(H_1\) lautet somit: Die Daten der Grundgesamtheit sind nicht normalverteilt. Wir wissen, dass die Stichproben aus einer gleichverteilten Grundgesamtheit stammen, also ist die Nullhypothese \(H_0\) falsch. Wir wollen nun untersuchen, wie oft die Nullhypothese \(H_0\) durch den Test tatsächlich verworfen wird. Welche qualitativen Ergebnisse erwarten wir? 9.5.4 Vergleich mit dem Kolmogorov-Smirnov-Test

Mithilfe unserer oben definierten Funktionen ist es nun sehr leicht, den Shapiro-Wilk-Test shapiro.test() durch den Kolmogorov-Smirnov-Test ks.test() zu ersetzen und beide Tests direkt zu vergleichen.

pKSTest <- function(n){ 
  stichprobe <- runif(n, min=-1, max=1) 
  h <- ks.test(stichprobe, "pnorm")
  return(h$p.value)
}

testKS <- function(n){
  replicateTest(n, FUN=pKSTest, nTest=1000, alpha=0.1)
}

9.6.1.4 Lineare Regression

Mithilfe linearer Regression können wir die Ausgleichsgerade in Abbildung 9.30 auch direkt berechnen. Dazu nutzen wir die mächtige Funktion lm():

mod_xy <- lm(y ~ 1 + x , data = dd)

Der Ausdruck y ~ x beschreibt eine Formel (formula) in R. Eine Formel ist ein eigener Datentyp; sie ist typischerweise von der Form y ~ model, wobei model aus einer Reihe von Variablen (Einflussgrößen) besteht, die die Größe y beschreiben und für die entsprechenden Modell-Parameter bestimmt werden sollen (wie zum Beispiel durch Kleinste-Quadrat-Schätzer der Regressionskoeffizienten für lineare Modelle). In der folgenden Tabelle, in der einfache lineare Modelle aufgelistet sind, sind die zu bestimmenden Modell-Parameter: \(\alpha\), \(\beta\) und \(\gamma\).

Formel Modell
y ~ x \(y = \beta x + \alpha\)
y ~ x - 1 \(y = \beta x\)
y ~ x, z \(y = \beta x + \gamma z + \alpha\)
